# 记忆自动进化系统 (Memory Auto-Evolution System)

> 让 Agent 自动从任务中学习，持续改进，无需人工干预

---

## 🎯 系统目标

1. **自动记录** — 每次任务后自动记录关键信息
2. **自动分析** — 从成功/失败中提取经验教训
3. **自动更新** — 更新记忆文件、技能文档、工作方式
4. **自动优化** — 基于使用数据持续改进

---

## 🔄 进化机制

### 机制 1: 任务后自动复盘

**触发**: 任务完成时 (成功/失败/取消)

**流程**:
```
任务完成
    ↓
分析结果 → 成功因素？失败原因？
    ↓
提取教训 → 什么该做？什么不该做？
    ↓
更新记忆 → daily → MEMORY.md
    ↓
更新文档 → SKILL.md / AGENTS.md
```

**自动记录内容**:
- 任务描述
- 执行过程关键步骤
- 结果（成功/失败）
- 耗时
- 遇到的问题
- 解决方案
- 经验教训

---

### 机制 2: 每日记忆整理

**触发**: 每天 23:00 (Cron)

**流程**:
```
读取今日 memory/YYYY-MM-DD.md
    ↓
提取关键事件
    ↓
分类整理
    ├─ 重要决策 → MEMORY.md
    ├─ 新技能 → 更新技能文档
    ├─ 工作方式改进 → AGENTS.md
    └─ 临时信息 → 归档或删除
    ↓
清理过期信息
```

---

### 机制 3: 技能自动优化

**触发**: 技能使用次数 > 10 次

**分析维度**:
- 最常用的参数/选项
- 常见错误及解决方案
- 最佳实践模式
- 性能瓶颈

**自动更新**:
- SKILL.md 添加 "常见问题" 章节
- 更新 "最佳实践" 示例
- 优化默认参数

---

### 机制 4: 周期性模式识别

**触发**: 每周日

**分析内容**:
- 本周任务类型分布
- 高频出现的问题
- 效率瓶颈
- 用户反馈模式

**输出**:
- 系统性改进建议
- AGENTS.md 更新
- 新的自动化机会

---

└── auto_evolution.py      # 自动进化脚本

## 🚀 快速启动

### 1. 安装自动进化脚本

```bash
# 创建自改进目录
mkdir -p ~/.openclaw/workspace/self-improving

# 复制脚本
cp auto_evolution.py ~/.openclaw/workspace/self-improving/

# 设置定时任务
python auto_evolution.py --setup-cron
```

### 2. 配置进化参数

编辑 `config/evolution.yaml`:
```yaml
evolution:
  # 任务后复盘
  post_task_analysis: true
  
  # 每日整理时间
  daily_cleanup: "23:00"
  
  # 技能优化阈值
  skill_optimization_threshold: 10
  
  # 周期性分析
  weekly_analysis: true
  weekly_analysis_day: "sunday"
  
  # 记忆保留策略
  retention:
    daily_memory_days: 30
    significant_events: permanent
    skill_lessons: permanent
```

### 3. 启动自动进化

```bash
# 手动运行一次
python auto_evolution.py --run-now

# 查看进化报告
python auto_evolution.py --report
```

## 📝 记忆更新规则

### 什么应该记住？

✅ **必须记录**:
- 重要决策及原因
- 失败的教训
- 新技能/工具的使用方法
- 用户明确要求的偏好
- 系统配置和依赖关系
- 项目关键信息

❌ **不必记录**:
- 临时查询的结果
- 公开可获取的信息
- 显而易见的事实
- 一次性任务细节
- 敏感信息（密码、密钥）

---

## 🔍 进化质量检查

### 自动检查项

- [ ] 每日记忆是否及时更新
- [ ] MEMORY.md 是否定期维护
- [ ] 技能文档是否过时
- [ ] 重复问题是否已记录解决方案
- [ ] 工作方式是否持续优化

### 人工审查点（每月）

- 检查自动更新的准确性
- 清理不再相关的记忆
- 验证技能文档的有效性
- 确认进化方向是否正确

---

## 💡 最佳实践

1. **写比想重要** — 不要依赖"记住"，要写下来
2. **定期回顾** — 每周查看 MEMORY.md，更新过时的信息
3. **结构化记录** — 使用时间戳、分类标签
4. **提取模式** — 从多个具体事件中提取通用规律
5. **持续迭代** — 记忆系统是活的，要不断优化

---

## 🔒 安全与隐私

- **敏感信息绝不自动记录** — 密码、API Key、私钥
- **分级存储** — 公开/内部/私密信息分开存储
- **定期审计** — 检查是否有敏感信息泄露到记忆文件
- **用户控制** — 用户可以随时查看、编辑、删除记忆
